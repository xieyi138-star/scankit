The barcode + OCR landscape is two separate products.

And neither one works well on its own.

💸 Dynamsoft / LEADTOOLS

$2,000+/year SDK licenses. Great for enterprises. Absurd for indie devs and startups.

🐛 ZXing (free, open-source)

Misses ~1 in 6 damaged barcodes. No OCR. No REST API. Java library, not a service.

🔀 OCR + barcode: always separate

Nobody combines them. You need pytesseract for text + pyzbar for barcodes. Two libraries, two pipelines, double the code.

📦 Cloudmersive (cloud-only)

The only combined offering. But cloud-only, no self-host option, per-call pricing adds up fast.

📋 Shipping labels, invoices, receipts

The most common use case — "read the tracking number AND the address" — has no single-call solution.

🖼️ Mobile photos: blurry, skewed

Phone pictures of barcodes fail on ZXing. No pre-processing, no rotation correction, no deblurring.

What ScanKit does

Two endpoints. Everything you need.

📸 /v1/decode

Upload an image (JPG, PNG, PDF page). Get back all barcodes, QR codes, and OCR text. One call.

🔧 /v1/generate

Generate QR codes, Code 128, EAN-13, UPC-A, Data Matrix, PDF417. Returns PNG or SVG. One line.

📐 Auto-rotation & deblur

Phone photo of a shipping label at a weird angle? We fix it before decoding. Works where ZXing fails.

🔌 REST, not an SDK

No language bindings. No native dependencies. No DLL hell. HTTP is the API. Works with curl, Python, Go, JS, anything.

🔐 Self-host or cloud

MIT-licensed open core. Run it yourself with Docker. Or use our hosted API at $9/month.

📊 Usage dashboard
